### Базовые настройки

1. **Настройка проекта:**
   - Создана новая директория для проекта и настроено виртуальное окружение.
   - Инициализирован новый проект Django внутри созданной директории.
   - Добавлен файл .gitignore для исключения ненужных файлов из репозитория.
   - Настроены правила ведения разработки по GitFlow:
     - Созданы ветки main и develop.
     - Для каждой домашней работы созданы отдельные ветки, например, feature/homework-1, feature/homework-2 и т.д.
   - Оформлен файл README.md с описанием проекта.

2. **Создание и настройка приложения:**
   - Создано новое приложение под названием catalog, зарегистрированное в настройках проекта (settings.py).
   - Создано новое приложение для блога под названием blog, также зарегистрированное в settings.py.
   - Настроена маршрутизация для нового приложения с добавлением соответствующих URL-адресов в файле urls.py.

3. **Создание моделей:**
   - В приложении catalog созданы модели Product и Category с базовыми полями.
   - В приложении blog создана модель BlogPost со следующими полями:
     - **title:** заголовок статьи.
     - **content:** содержимое статьи.
     - **preview_image:** превью (изображение).
     - **created_at:** дата создания.
     - **is_published:** признак публикации (булевое поле).
     - **views_count:** количество просмотров.

4. **Реализация CRUD для модели BlogPost:**
   - Реализован полный CRUD для модели BlogPost с использованием классовых представлений (CBV):
     - Создание, чтение, обновление и удаление статей.

5. **Миграции:**
   - Перенесено отображение моделей в базу данных с помощью миграций:
     - Созданы и применены миграции для новых моделей.
     - Все выполненные миграции добавлены в коммит и отправлены в удаленный репозиторий на GitHub.

6. **Модификация контроллеров:**
   - Увеличение счетчика просмотров: при открытии отдельной статьи увеличивается счетчик просмотров.
   - Фильтрация опубликованных статей: выводится в список статей только те, которые имеют положительный признак публикации.
   - Перенаправление после редактирования: после успешного редактирования записи пользователь перенаправляется на просмотр этой статьи.
   - Отправка уведомления: когда статья достигает 100 просмотров, отправляется поздравительное письмо администратору.

7. **Создание шаблонов:**
   - Подготовлены два HTML-шаблона: home.html для домашней страницы и contacts.html для страницы с контактной информацией, стилизованные с использованием Bootstrap.
   - Созданы шаблоны для отображения списка статей и отдельной статьи блога.

8. **Админка:**
   - Создан суперпользователь.
   - Модели Product, Category и BlogPost зарегистрированы в админке:
     - Для каждой модели выведены соответствующие поля в списке.
     - Настроены фильтрация и поиск по полям.

9. **Фикстуры и кастомные команды:**
   - Сформированы фикстуры для моделей Category, Product и BlogPost.
   - Создана кастомная команда для добавления тестовых продуктов и блоговых записей с удалением существующих данных перед добавлением новых.

10. **Дополнительные функции:**
    - В контроллер отображения главной страницы добавлена выборка последних 5 созданных продуктов с выводом их в консоль.
    - Создана модель для хранения контактных данных, данные которой выводятся через соответствующий контроллер.

### Аутентификация пользователей

1. **Регистрация пользователя:**
   - Создана форма для регистрации пользователя с полями электронной почты и пароля.
   - Добавлена логика регистрации в представлении, позволяющая пользователю создать учетную запись.
   - Реализована отправка приветственного письма пользователю после регистрации.

2. **Авторизация пользователя:**
   - Настроены форма и представление для авторизации пользователя по электронной почте и паролю.
   - Закрыт доступ к страницам, связанным с управлением продуктами, для анонимных пользователей:
     - Убедились, что только авторизованные пользователи могут просматривать, создавать, изменять и удалять продукты.
     - Общедоступной осталась только страница просмотра списка товаров.
   - Добавлен интерфейс редактирования профиля пользователя.

3. **Группы и разрешения:**
   - Создана группа для роли «Модератор продуктов» и настроены необходимые разрешения:
     - Добавлено к модели продукта поле, определяющее статус публикации (BooleanField или CharField).
     - Созданы фикстуры или кастомная команда для создания групп и их прав.

4. **Кастомные права:**
   - Добавлены кастомные права к модели продукта:
     - can_unpublish_product — может отменять публикацию продукта.
   - Настроены проверки в контроллерах и шаблонах для использования этих прав.
   - Реализована проверка прав у пользователя при попытке отменить публикацию или удалить продукт.

5. **Владельцы продуктов:**
   - Реализовано решение, что редактирование и удаление продукта доступно только его владельцу:
     - Добавлено поле владельца (owner) к модели продукта, связанное с моделью пользователя через ForeignKey.
     - При создании нового продукта поле owner автоматически заполняется текущим авторизованным пользователем.
     - Настроены проверки в контроллерах, чтобы только владелец мог редактировать и удалять продукт.
     - Удалять продукты может не только владелец, но и модератор.
     - Обновлены шаблоны, чтобы кнопка редактирования и удаления отображалась только для владельца продукта.

6. **Группа «Контент-менеджер»:**
   - Создана группа для роли «Контент-менеджер», позволяющая управлять публикациями в блоге.
   - Реализованы проверки, чтобы обычный пользователь или модератор из другого отдела не мог изменять раздел блога.
